Box Model

In web development, the CSS box model refers to how HTML elements are modeled in browser engines and how dimension of those HTML elements are derived from CSS properties. It is a fundamental concept for the composition of HTML webpages. The guidelines of the box model are described by web standards World Wide Web Consortium (W3C) specifically the CSS Working Group. For much of late-1990s and early 2000s there had been non-standard compliant implementations of the box model in the mainstream browsers. With the advent of CSS2, which introduced the box-sizing property, the problem had mostly been resolved.

Box Model Pratybos

Sukurkite 60x60 pikselių dydžio kvadratus (bet kokios spalvos) kurie turėtų:

  • 10px margin iš viršaus ir apačios
  • 10px margin iš kairės
  • 10px margin iš visų pusių, išskyrus dešinės
  • padding iš visų pusių po 10px
  • 10px margin viršuje, 10px padding iš kairės ir apačios. Uždėkite mė sienelę
  • 10px visose pusėsė, 10px padding viršuje ir apačioje
  • 10px keirėje pusėsė, 10px padding dešinėje. Uždėkite mė sienelę.

Position

Position dokumentacija ir pavyzdys

Static

Tai toks pozicionavimo būdas, kuomet elementai išdeliojami lotyniško teksto rašymo tvarka. Iš kairės į dešinę, o kuomet elementai netelpa, persikelia į sekančią eilutę.

Relative

Tai toks pozicionavimo būdas, kuomet elementas yra traktuojamas lyg static, tačiau gali būti paslinktas naudojant CSS saybes: top, right, bottom, left. Naudojant šias savybes jis paslenkamas nuo jo menamos static pozicijos.

top: 1rem
left: 1rem
right: 1rem
bottom: 1rem

Absolute

Tai toks pozicionavimo būdas, kuomet elementas ieško tėvinės hierarchijos šakoje pirmo sutikto ne static elemento. Suradęs jį, atsispiria nuo jo kontūrų naudojant savybes: top, right, bottom, left. Jeigu tėvinės hierarchijos šakoje neranda elemento su ne static pozicionavimu, atsispiria nuo pradinio puslapio užkrovimo lango viewport'o.

Svarbus tekstas

top, right
bottom, left

Sticky

Tai toks pozicionavimo būdas, kuomet elementas ieško tėvinės hierarchijos šakoje pirmo sutikto ne static elemento. Suradęs jį, atsispiria nuo jo kontūrų naudojant savybes: top, right, bottom, left ir išlieka toje pačioje pozicijoje. Jeigu tėvinės hierarchijos šakoje neranda elemento su ne static pozicionavimu, atsispiria ir išlieka toje pačioje pozicijoje nuo pradinio puslapio užkrovimo lango viewport'o.
